<!-- Title: EADDRINUSE on every activation under WSL2 mirrored networking -->

## Describe the bug

The extension fails to activate with `EADDRINUSE` on every single attempt when running under WSL2 with `networkingMode=Mirrored` in `.wslconfig`. This is 100% reproducible — it never succeeds.

The internal `get-port` implementation (`ne()` → `te()`) finds a free port by binding to port 0, reading the assigned port, then closing the socket. The extension then creates a new server and tries to `.listen()` on that port. Under WSL2 mirrored networking, closing a socket bound to an ephemeral port leaves a kernel-level phantom listener (visible via `ss -tlnp` as `*:PORT` with no owning process) for ~1 second. The re-bind always fails during this window.

This affects both the `bt()` (child_process) and `St()` (terminal) code paths.

I'd like to submit a PR for this — the fix is to listen on port 0 directly and keep the server instance instead of closing and reopening (see suggested fix below).

## Expected behavior

The extension should activate and load the Vitest config without errors, regardless of WSL networking mode.

## Reproduction

Minimal repro of the underlying port issue (run inside WSL2 with `networkingMode=Mirrored`):

```js
const net = require('net');
let fail = 0, ok = 0;
function trial(n) {
  if (n <= 0) { console.log(ok + ' ok, ' + fail + ' fail'); return; }
  const s = net.createServer();
  s.listen(0, () => {
    const port = s.address().port;
    s.close(() => {
      const s2 = net.createServer();
      s2.listen(port, () => { ok++; s2.close(() => trial(n-1)); })
        .on('error', () => { fail++; trial(n-1); });
    });
  });
}
trial(10);
// Mirrored WSL2: 0 ok, 10 fail
// NAT WSL2 or native Linux: 10 ok, 0 fail
```

To reproduce the extension failure: have any project with a `vitest.config.ts`, run VS Code Remote - WSL with `networkingMode=Mirrored`, and activate the extension.

## Suggested Fix

Instead of using get-port (find port → close → re-listen), create the server and listen on port 0 directly, keeping the same server instance:

```js
// Before (broken under WSL2 mirrored networking):
const port = await getPort();
const server = createServer().listen(port);

// After (works everywhere):
const server = await new Promise((resolve, reject) => {
  const s = createServer();
  s.on('error', reject);
  s.listen(0, () => resolve(s));
});
const port = server.address().port;
```

This eliminates the close-and-reopen race entirely. The server stays bound to the OS-assigned port without ever releasing it.
